# parallel-file-creation スキル v2.0

> 複数の専門家が、それぞれの役割に応じたファイルを並列作成する

## 概要

このスキルは、開発チームのように複数の専門家（ペルソナ）が
それぞれの専門分野のファイルを並列で作成します。

```
┌─────────────┬─────────────┬─────────────┐
│  開発者     │  テスター   │  ライター   │
│ (技師)      │ (検分役)    │ (記録係)    │
├─────────────┼─────────────┼─────────────┤
│ auth.ts     │ auth.test.ts│ AUTH.md     │
│ コード実装  │ テスト作成  │ 仕様書作成  │
└─────────────┴─────────────┴─────────────┘
         ↓ 並列実行 ↓
      3つのファイルが同時に完成
```

## ペルソナ一覧

| ペルソナ | 日本語名 | 担当ファイル | 専門 |
|----------|----------|-------------|------|
| senior_engineer | 熟練の技師 | *.ts, *.tsx, *.py | コード実装 |
| technical_writer | 記録係 | *.md, README | ドキュメント |
| analyst | 軍師 | 分析レポート, 設計書 | 調査・分析 |
| ui_designer | 意匠師 | *.css, *.scss | デザイン |
| tester | 検分役 | *.test.ts, *.spec.ts | テスト |
| devops | 陣地構築師 | Dockerfile, CI/CD | インフラ |

## 使用例

### 例1: 機能開発（コード + テスト + ドキュメント）

```
ユーザー認証機能を作成してください：
- 開発者: lib/auth.ts（JWT認証ロジック）
- テスター: lib/auth.test.ts（ユニットテスト）
- ライター: docs/AUTH.md（仕様書）
```

**実行結果:**
```
足軽1 (熟練の技師) → lib/auth.ts 作成
足軽2 (検分役)     → lib/auth.test.ts 作成
足軽3 (記録係)     → docs/AUTH.md 作成
```

### 例2: UI機能開発

```
ダークモード機能を実装：
- 開発者: components/ThemeProvider.tsx
- デザイナー: styles/theme.css
- ライター: docs/THEME.md
```

### 例3: インフラ構築

```
CI/CD環境を構築：
- DevOps: .github/workflows/ci.yaml
- ライター: docs/DEPLOY.md
```

### 例4: フルスタック機能

```
商品一覧ページを作成：
- 開発者: pages/products.tsx（ページコンポーネント）
- 開発者: lib/api/products.ts（API呼び出し）
- デザイナー: styles/products.css（スタイル）
- テスター: __tests__/products.test.tsx（テスト）
- ライター: docs/PRODUCTS.md（仕様書）
```

## 実行フロー

```
1. 将軍: タスク受領・分析
   └─ 「認証機能を作れ」→ 3つのタスクに分解

2. 家老: ペルソナ別にタスク分配
   ├─ task_001.yaml → 足軽1 (senior_engineer)
   ├─ task_002.yaml → 足軽2 (tester)
   └─ task_003.yaml → 足軽3 (technical_writer)

3. 足軽: 並列でファイル作成
   ├─ 足軽1: auth.ts を専門家として実装
   ├─ 足軽2: auth.test.ts をテスト観点で作成
   └─ 足軽3: AUTH.md を分かりやすく記述

4. 家老: 報告集約・ダッシュボード更新

5. 将軍: 結果報告 + スキル評価
```

## ポイント

### 専門性の活用
各足軽は割り当てられたペルソナの専門家として振る舞います。
開発者はコードの品質を、テスターはテストカバレッジを、
ライターは可読性を重視してファイルを作成します。

### 最小権限原則
各足軽は自分の担当ファイルのみ作成・編集できます。
他の足軽のファイルに干渉することはありません。

### 並列実行
依存関係がないタスクは同時に実行されます。
3つのファイルを順番に作成するより高速です。

## 制約事項

- ファイル間に依存関係がある場合は順序を考慮
- 1つのファイルを複数の足軽が編集することは不可
- ペルソナに適さないファイルタイプは非推奨
